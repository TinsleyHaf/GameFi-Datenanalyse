# Contributing to GameFi-Datenanalyse

Thank you for considering contributing to **GameFi-Datenanalyse**! We appreciate your interest in helping us improve our project. Here are some guidelines to get you started:

## Contribution Guidelines

1. **Fork the repository**: Click on the "Fork" button at the top right corner of the repository page to create a copy of the repository in your GitHub account.

2. **Clone the repository**: Clone your forked repository to your local machine by running:
   ```bash
   git clone https://github.com/YOUR_USERNAME/GameFi-Datenanalyse.git
   ```
   Replace `YOUR_USERNAME` with your GitHub username.

3. **Create a new branch**: Before making changes, create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/YourFeatureName
   ```

4. **Make your changes**: Implement your feature or fix any bugs in your local repository.

5. **Commit your changes**: After making your changes, commit them with a clear and concise message:
   ```bash
   git commit -m "Add/Update: a brief description of your changes"
   ```

6. **Push to GitHub**: Push your changes to your forked repository:
   ```bash
   git push origin feature/YourFeatureName
   ```

7. **Open a pull request**: Navigate to the original repository on GitHub and click on the **Pull Requests** tab. Click on **New Pull Request**, select your branch, and submit the request. Please include a description of the changes you made and why they should be merged.

## Code Standards
- Follow standard coding conventions for consistency.
- Write clear and explanatory comments in your code.

## Additional Notes
- Make sure all new features are covered by tests.
- Please keep your commits focused and single-purpose.

We look forward to your contributions! If you have any questions or need assistance, feel free to reach out.

Happy Coding!
